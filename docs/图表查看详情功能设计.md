# 图表查看详情功能设计文档

## 1. 功能概述

### 1.1 功能描述
在所有图表（折线图、饼图、柱状图、雷达图等）中，点击任意数据元素时，弹出一个详情对话框，显示该数据点对应的具体工单列表。

### 1.2 核心价值
- **数据透明度**：让用户看到图表背后的真实数据
- **交互深度**：从宏观统计到微观明细的无缝过渡
- **问题定位**：快速定位具体问题工单
- **决策支持**：基于真实数据做决策

---

## 2. 详细需求分析

### 2.1 适用图表类型

| 图表类型 | 点击元素 | 筛选逻辑 | 示例 |
|---------|---------|---------|------|
| **折线图** | 数据点/折线 | 时间范围 + 指标 | 点击"2026-03月"的创建工单点 → 显示3月所有工单 |
| **柱状图** | 柱子 | 维度值 | 点击"杨洪波"柱子 → 显示杨洪波的所有工单 |
| **饼图** | 扇区 | 维度值 | 点击"需求问题"扇区 → 显示所有需求问题工单 |
| **雷达图** | 数据点 | 维度值 | 点击某个经办人的数据点 → 显示该经办人工单 |
| **散点图** | 数据点 | 具体工单 | 点击某个点 → 显示该工单详情 |
| **热力图** | 热力格子 | 双维度筛选 | 点击某个格子 → 显示符合双维度的工单 |
| **漏斗图** | 漏斗层级 | 状态值 | 点击"待分析"层 → 显示所有待分析工单 |
| **桑基图** | 节点/连线 | 维度值/路径 | 点击某个节点 → 显示相关工单 |

#### 功能特性
1. **标题栏**
   - 显示筛选条件（如"经办人 = 杨洪波"）
   - 显示工单总数
   - 关闭按钮

2. **工具栏**
   - 搜索框：搜索关键字、概要等
   - 状态筛选下拉框
   - 排序选择：按时间/时长/状态排序
   - 导出Excel按钮

3. **数据表格**
   - 列：关键字、概要、经办人、状态、创建时间、处理时长、紧急程度
   - 行：每行一个工单
   - 点击行：跳转到工单详情页（可选）

4. **分页**
   - 每页20条
   - 显示总页数
   - 上一页/下一页按钮

### 2.3 交互流程

#### 流程图
```
用户点击图表元素
    ↓

    ↓
```

#### 3.2.3 useChartClick.ts - 图表点击逻辑
```typescript
// src/composables/useChartClick.ts

import type { Ticket } from '@/types'
import { useDataStore } from '@/stores/dataStore'

export interface ChartClickData {
  componentType: string
  seriesType: string
  seriesName: string
  name: string
  value: number | number[]
  dataIndex: number
  data: any
}

export interface FilterCondition {
  field: string      // 筛选字段
  operator: string   // 操作符：eq, in, between
  value: any         // 筛选值
  description: string // 筛选描述
}

export function useChartClick() {
  const dataStore = useDataStore()

  /**
   * 根据图表点击数据构建筛选条件
   */
  function buildFilterCondition(
    chartType: string,
    clickData: ChartClickData,
    chartConfig?: any
  ): FilterCondition[] {
    const conditions: FilterCondition[] = []

    switch (chartType) {
      case 'line':
        // 折线图：通常按时间筛选
        if (clickData.name) {
          conditions.push({
            field: 'createDate',
            operator: 'month',
            value: clickData.name, // 如 "2026-03"
            description: `创建月份 = ${clickData.name}`
          })
        }
        break

      case 'bar':
        // 柱状图：按维度值筛选
        if (clickData.name) {
          // 根据图表配置判断筛选字段
          const field = chartConfig?.filterField || 'assignee'
          conditions.push({
            field,
            operator: 'eq',
            value: clickData.name,
            description: `${getFieldDisplayName(field)} = ${clickData.name}`
          })
        }
        break

      case 'pie':
        // 饼图：按维度值筛选
        if (clickData.name) {
          const field = chartConfig?.filterField || 'customerIssueType'
          conditions.push({
            field,
            operator: 'eq',
            value: clickData.name,
            description: `${getFieldDisplayName(field)} = ${clickData.name}`
          })
        }
        break

      case 'scatter':
        // 散点图：可能需要多维度筛选
        if (clickData.data && clickData.data.length >= 3) {
          // 假设data[2]是工单key
          conditions.push({
            field: 'key',
            operator: 'eq',
            value: clickData.data[2],
            description: `工单 = ${clickData.data[2]}`
          })
        }
        break

      case 'heatmap':
        // 热力图：双维度筛选
        // X轴和Y轴的值
        const xValue = chartConfig?.xAxisData?.[clickData.data[0]]
        const yValue = chartConfig?.yAxisData?.[clickData.data[1]]

        if (xValue && yValue) {
          conditions.push({
            field: chartConfig.xField,
            operator: 'eq',
            value: xValue,
            description: `${getFieldDisplayName(chartConfig.xField)} = ${xValue}`
          })
          conditions.push({
            field: chartConfig.yField,
            operator: 'eq',
            value: yValue,
            description: `${getFieldDisplayName(chartConfig.yField)} = ${yValue}`
          })
        }
        break

      case 'funnel':
        // 漏斗图：按状态筛选
        if (clickData.name) {
          conditions.push({
            field: 'status',
            operator: 'eq',
            value: clickData.name,
            description: `状态 = ${clickData.name}`
          })
        }
        break

      case 'radar':
        // 雷达图：按经办人筛选
        if (clickData.name) {
          conditions.push({
            field: 'assignee',
            operator: 'eq',
            value: clickData.name,
            description: `经办人 = ${clickData.name}`
          })
        }
        break
    }

    return conditions
  }

  /**
   * 根据筛选条件过滤工单
   */
  function filterTickets(conditions: FilterCondition[]): Ticket[] {
    return dataStore.filteredData.filter(ticket => {
      return conditions.every(condition => {
        const fieldValue = ticket[condition.field as keyof Ticket]

        switch (condition.operator) {
          case 'eq':
            return fieldValue === condition.value

          case 'in':
            return Array.isArray(condition.value) &&
                   condition.value.includes(fieldValue)

          case 'between':
            if (Array.isArray(condition.value) && condition.value.length === 2) {
              const val = fieldValue as any
              return val >= condition.value[0] && val <= condition.value[1]
            }
            return false

          case 'month':
            // 月份筛选
            if (fieldValue instanceof Date) {
              const month = format(fieldValue, 'yyyy-MM')
              return month === condition.value
            }
            return false

          default:
            return true
        }
      })
    })
  }

  return {
    buildFilterCondition,
    filterTickets
  }
}

function getFieldDisplayName(field: string): string {
  const map: Record<string, string> = {
    assignee: '经办人',
    customerIssueType: '客户问题类型',
    domain: '领域模块',
    status: '状态',
    priority: '紧急程度',
    projectName: '项目名称',
    createDate: '创建日期'
  }
  return map[field] || field
}
```

### 4.2 技术难点

#### 难点1：不同图表的点击数据解析
- **问题**：不同图表类型的点击数据格式不同
- **解决方案**：在 useChartClick 中针对每种图表类型单独处理

#### 难点2：大数据量性能优化
- **问题**：如果筛选结果有上千条工单，表格渲染慢
- **解决方案**：
  - 使用虚拟滚动（el-table-v2）
  - 分页展示（每页20条）
  - 前端缓存筛选结果

#### 难点3：图表配置信息传递
- **问题**：需要知道图表的筛选字段是什么
- **解决方案**：在图表配置中添加 meta 信息
